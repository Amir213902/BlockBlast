# 📱 PWA Setup Guide - BlockBlast

BlockBlast теперь является полноценным Progressive Web App (PWA) приложением! 🚀

## Что такое PWA?

PWA позволяет:
- ✅ **Установить приложение** на главный экран без AppStore
- ✅ **Работать офлайн** благодаря Service Worker
- ✅ **Быстрая загрузка** с кешированием
- ✅ **Push-уведомления** (опционально)
- ✅ **Полноэкранный режим** как нативное приложение

## 📋 Что добавлено

### 1. **manifest.json** (`/public/manifest.json`)
- Описание приложения
- Иконки (192×192 и 512×512)
- Tema цвета и background
- Ориентация экрана

### 2. **Service Worker** (`/public/sw.js`)
- Автоматическое кеширование статических файлов
- Работа офлайн
- Обновление кеша при новых версиях

### 3. **Meta-теги в HTML** (`index.html`)
- PWA-совместимые мета-теги
- Apple iOS поддержка
- Регистрация Service Worker

### 4. **Иконка приложения** (`/public/logo.svg`)
- Красивая SVG иконка с блоками
- Адаптивная для разных размеров

## 🎮 Как установить приложение

### На Android:
1. Откройте приложение в Chrome/Firefox
2. Нажмите меню (⋮) → **Установить приложение**
3. BlockBlast добавится на главный экран

### На iOS:
1. Откройте приложение в Safari
2. Нажмите поделиться → **На экран "Домой"**
3. BlockBlast появится как иконка

### На рабочего стола (Chrome):
1. Откройте адрес в Chrome
2. Нажмите иконку установки в адресной строке
3. Приложение будет доступно как обычное приложение

## 🔄 Как работает кеширование

**Первый визит:**
- Загружаются все файлы
- Сохраняются в кеш браузера

**Последующие визиты:**
- Приложение загружается из кеша (быстро!)
- В фоне обновляется кеш с новыми версиями
- Работает даже без интернета

## 📝 Файлы PWA

```
frontend/
├── index.html                 # Meta-теги для PWA
├── public/
│   ├── manifest.json         # Конфигурация приложения
│   ├── sw.js                 # Service Worker
│   └── logo.svg              # Иконка приложения
```

## 🚀 Тестирование

### Проверить в DevTools (Chrome):
1. Откройте **DevTools** (F12)
2. Перейдите **Application** → **Manifest**
3. Должны быть видны все поля из `manifest.json`
4. Перейдите **Application** → **Service Workers**
5. Должно быть status **activated**

### Проверить офлайн режим:
1. DevTools → **Network** → **Offline**
2. Перезагрузите страницу
3. Приложение должно работать из кеша!

## 🔧 Как изменить параметры

### Изменить имя приложения:
Отредактируйте `manifest.json`:
```json
{
  "name": "Новое имя",
  "short_name": "Короткое имя"
}
```

### Изменить иконку:
Замените `/public/logo.svg` на новый файл и обновите пути в `manifest.json`

### Изменить цвета темы:
```json
{
  "theme_color": "#000D4D",
  "background_color": "#000D4D"
}
```

## 📊 Проверка готовности PWA

Используйте **Lighthouse** (встроен в Chrome DevTools):
1. DevTools → **Lighthouse**
2. Запустите audit для **PWA**
3. Должны быть зелёные галочки ✅

## 🐛 Возможные проблемы

**Приложение не устанавливается?**
- Убедитесь, что используется HTTPS (не HTTP)
- Service Worker должен быть активирован
- Проверьте `manifest.json` на ошибки

**Кеш не обновляется?**
- Очистите кеш вручную: DevTools → Application → Clear storage
- Перезагрузите страницу

**На iOS не видно иконки?**
- Добавьте иконку формата PNG размером 180×180px
- Используйте `<link rel="apple-touch-icon" href="/icon.png">`
